# Cortex Charter

Remram Cortex exists to turn memory into knowledge.

OpenClaw provides deterministic runtime execution, structured tool use, and local control. Cortex addresses the missing layer above that runtime: durable knowledge that can be extracted, reinforced, reconciled, and promoted over time.

It is infrastructure, not a plugin. The goal is knowledge maturation, not just memory recall.

## Core Shift

Evidence captures what happened.

Knowledge extracts what mattered.

Cortex exists so experience does not remain trapped as raw history or prompt residue.

## Why Cortex Exists

### Knowledge Over Sessions

Longer transcripts are not the answer. Cortex stores normalized knowledge objects with explicit governance fields, typed retrieval signals, relationships, confidence, and source provenance so durable memory survives compaction and pruning.

### Token Efficiency

Retrieval should be selective, bounded, and intentional. Hard governance filters should constrain the candidate set before semantic scoring. Only eligible, high-value knowledge should be injected into runtime context. This keeps prompts smaller, cloud dependence lower, and execution easier to govern.

### Reflection

Reflection is the path from interaction to knowledge. It should focus on deltas that matter rather than repeatedly storing the same already-known facts.

### Artifact Intake

Imported documents and images should not stay opaque attachments. Cortex should parse them through one artifact-intake flow so source-linked memory can be created before the user is asked to do manual summarization.

### Bootstrap

Historical conversations should be able to initialize Cortex through read-only ingestion. The goal is not transcript import for its own sake, but waking the system up with structured continuity and an already-useful knowledge base.

### Dream

Dream is the offline maturation routine. It reconciles contradictions, strengthens recurring patterns into principles, and identifies knowledge that deserves promotion into durable artifacts.

### Knowledge As An Asset

High-value knowledge should be able to mature into durable outputs such as decision records, narratives, plans, and other reviewable artifacts. Cortex is not just a retrieval layer; it is the system that helps experience compound into reusable assets.

### Local-First, Not Isolated

Cortex remains local-first and inspectable. Directionally, knowledge should also be able to participate in explicit, permissioned sharing models without giving up ownership or collapsing into a generic shared memory pool. The exact sharing model remains an active design area.

## Adoption Direction

The design assumes enthusiast-first adoption. Infrastructure integrity takes priority over convenience wrappers, even if onboarding is slower in the short term.
