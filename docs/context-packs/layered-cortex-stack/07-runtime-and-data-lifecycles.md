# Runtime And Data Lifecycles

## Flow 1: Live Session To Layer 2

1. OpenClaw runs the session.
2. the transcript and runtime events remain the evidence surface.
3. QMD supports hot retrieval and notion storage.
4. turn-end and session-end processing emit typed semantic outputs.
5. Layer 2 continuity is assembled from session state, QMD, and those outputs.

Later, Mamba can improve this same flow by providing a better always-on signal source.

## Flow 2: Many Threads To One Emerging Idea

1. several threads contain fragments of one emerging concept
2. Layer 3 identifies the semantic relationship
3. Layer 4 holds the evolving workspace body
4. reflection and Dream continue organizing that workspace
5. the workspace may later become a promotion candidate

## Flow 3: Layer 2 To Layer 3

1. a notion is created or updated in QMD
2. it may be used for tentative continuity under tighter retrieval rules
3. reflection evaluates it
4. high-value meaning promotes into Layer 3 durable memory when warranted

## Flow 4: External Reference Material

1. a reference is introduced
2. Layer 4 stores summaries, linked records, and operational knowledge
3. Layer 3 stores only durable meaning that matters
4. the reference is not automatically turned into canon

## Flow 5: Workspace To Canonical Artifact

1. a Layer 4 workspace matures
2. Layer 3 support and relationship structure indicate readiness
3. redraft and publication are initiated
4. Layer 5 receives a canonical artifact only when canon is warranted
5. once promoted, Layer 5 becomes the canonical source for that artifact
6. later meaningful Layer 5 revisions can trigger re-ingestion into Layer 4
7. Layer 3 summaries, support, and semantic links are reconciled as needed

## Flow 6: Evidence Retention

1. transcripts stay hot for a bounded recent window
2. older evidence moves to cold storage
3. compact metadata and semantic checkpoints remain hot longer

## Big Picture

The lifecycle rule is:

- hot working continuity in Layer 2
- durable meaning in Layer 3
- operational bodies in Layer 4
- canonical publication in Layer 5 only when applicable
- Mamba later as a hardening and optimization supercharge rather than a prerequisite for Phase 1
