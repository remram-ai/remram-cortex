# Test Plan

## Core Scenarios

1. Base trust and reconciliation work without Intuition.
2. Session-end reconciliation resolves tentative QMD notions correctly.
3. Reflection keeps QMD lean through prune, merge, demote, and expire behavior.
4. Nightly reconciliation improves trust through merge, supersession, or invalidation.
5. Dream-like maintenance uses Layer 3 relationships to identify Layer 4 grouping and promotion candidates.

## Failure Checks

- Intuition should not silently become the memory authority
- approval-sensitive changes should not bypass review
- nightly maintenance should not break provenance links

## Evidence Of Completion

- one documented later Intuition-trigger case
- one session-end promotion or rejection case
- one QMD cleanup case
- one nightly consolidation or stale-support cleanup case
