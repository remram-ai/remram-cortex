# Reflection, Oversight, And Reconciliation

## Reflection

Reflection is the near-time path.

It is allowed to:

- update notions
- prune notions
- merge notions
- demote stale notions
- keep QMD lean
- update Layer 4 workspaces
- use Layer 3 relationships to organize Layer 4 work
- detect promotion candidates

## Oversight

Oversight observes the relevant semantic products and intervenes when trust, policy, or approval posture requires it.

It may consume:

- notion activity
- high-signal stream events
- promotion candidates

## Reconciliation

Reconciliation is how tentative working-memory candidates become trusted durable memory.

Fast cross-thread continuity is allowed.

Silent authority is not.

## Dream

Dream is the slower consolidation path.

It is allowed to:

- revisit Layer 3 relationship structure
- consolidate idea clusters
- detect workspace merges
- detect promotion readiness
- harden support and supersession

Both Reflection and Dream use Layer 3 to help organize Layer 4.

Neither should turn Layer 3 into a workspace body store.
