# Cortex Charter

Remram Cortex exists to turn runtime evidence and source artifacts into durable, inspectable, revision-aware knowledge.

It does this through a layered architecture instead of forcing every job through one memory substrate.

## Core Commitments

Cortex is committed to:

- explicit policy composition
- bounded working-memory assembly
- durable semantic memory with provenance and reconciliation
- decomposed knowledge that is operationally useful but not mistaken for canonical truth
- canonical artifacts that remain reviewable and publication-grade

## System Boundary

OpenClaw executes.

Cortex:

- decides what should persist
- decides how evidence becomes notions and then durable memory
- decides how decomposed knowledge and canonical artifacts stay aligned
- decides how oversight, reconciliation, and publication work

## Layer Commitments

### Policy

Policy is custom.

It includes:

- role and mode selection
- tool policy
- safety and approval rules
- prompt-budget discipline
- mutable preference-policy through governed workflows

### Working Memory

Working memory stays OpenClaw-native by default.

Cortex augments it through:

- policy-aware context assembly
- semantic checkpoint production
- Mamba-style compression

### Durable Memory

Durable memory is the semantic authority.

It receives:

- staged notions
- reconciled beliefs
- support and provenance links
- contradiction and supersession updates

### Decomposed Artifact Knowledge

This layer is allowed to move fast.

It exists for:

- retrieval
- decomposition
- source-linked slices
- operational knowledge ahead of publication

### Canonical Artifacts

This layer stays authoritative for publication-grade source truth.

It is where:

- Git-backed artifacts
- provider-backed authoritative documents
- reviewed redrafts

belong.

## Why This Exists

Without Cortex, systems drift toward:

- transcript-as-memory
- prompt-as-policy
- search-index-as-truth
- derived slices masquerading as canonical documents

Cortex exists to stop that drift.
