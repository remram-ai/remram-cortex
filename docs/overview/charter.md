# Cortex Charter

Remram Cortex exists to turn evidence into durable, usable knowledge through a layered architecture.

OpenClaw provides runtime execution, tool use, and session orchestration.

Cortex addresses the layer above that runtime:

- working continuity that does not bloat prompts
- durable memory that survives sessions
- artifact-backed knowledge that remains revision-aware
- publication workflows that let working knowledge mature into canonical artifacts

It is infrastructure, not a plugin.

The goal is not just memory recall.

The goal is coordinated knowledge maturation across layers.

## Core Shift

Evidence captures what happened.

Working memory supports what matters now.

Durable memory retains what should persist.

Knowledge preserves what should be inspectable, retrievable, and publishable.

Cortex exists so experience does not remain trapped as raw history, prompt residue, or opaque documents.

## Why Cortex Exists

### Layer Separation

The system needs explicit separation between:

- policy
- working state
- durable memory
- derived retrieval knowledge
- canonical artifact truth

These are not the same thing and should not be forced through one substrate.

### Knowledge Over Session Dependence

Longer transcripts are not the answer.

The system needs structured durable memory that can be updated, corrected, supported, and retrieved deliberately instead of relying on ever-growing session history.

This preserves the original `memory versus context` distinction:

- context helps the system operate now
- durable memory preserves what should survive

### Bounded Runtime Injection

Prompt assembly should be selective, bounded, and intentional.

The system should inject compact policy, working continuity, and durable memory by default, and reach into deeper knowledge only when needed.

This keeps the original `bounded retrieval` principle intact.

It also preserves the earlier idea that retrieval quality should be governed before large amounts of context are injected.

### Governance And Retrieval Signals

Cortex should still use explicit semantic and governance structure inside durable memory and retrieval.

Compatible earlier concepts remain valid here:

- governance fields
- typed signals
- semantic signature

The layered shift does not remove them.

It clarifies that they belong mainly in durable memory and retrieval orchestration, not in hot working state.

### Reflection

Reflection is how evidence becomes memory and how evidence also changes artifact understanding.

It should branch one evidence feed into:

- durable-memory updates
- artifact-impact updates

without forcing everything into one hot path.

This preserves the earlier reflection concept while making its outputs more explicit.

### Dream And Maintenance

Dream-like maintenance still matters.

It remains the slower reconciliation layer that:

- consolidates recurring patterns
- checks contradictions
- strengthens or weakens durable memory over time
- helps determine what deserves promotion or publication

### Artifact-Backed Knowledge

Imported documents and internal artifacts should not remain opaque attachments.

They should become revision-aware, decomposed knowledge that can be searched, cited, and later republished into canonical form.

This keeps the original `artifact intake` direction intact, but places it inside the knowledge layer instead of treating it as just another kind of memory write.

### Publication And Redraft

Working knowledge is allowed to move ahead of canonical artifacts during active work.

Canonical truth catches up through redraft, review, publication, and re-ingestion.

That publication loop is part of Cortex, not an external afterthought.

This is also where `artifact promotion` now fits most naturally.

Promotion is still the path from evolving understanding into durable reviewable outputs.

### Bootstrap

Historical conversations and source material should still be able to initialize Cortex through controlled ingestion.

The point remains the same as before:

- wake the system up with useful continuity
- do not confuse raw historical import with durable truth

### Knowledge As An Asset

High-value knowledge should mature into durable outputs such as plans, records, narratives, and other reviewable artifacts.

Cortex is not just the memory of the system.

It is the compounding knowledge layer of the system.

### Local-First, Not Isolated

Cortex remains local-first and inspectable.

Directionally, knowledge should still be able to participate in explicit, permissioned sharing models without collapsing into a generic pooled memory system.

## Adoption Direction

The design still assumes enthusiast-first adoption.

Infrastructure integrity, inspectability, and clean authority boundaries take priority over convenience wrappers, even if onboarding is slower in the short term.

## Boundary Reminder

This charter explains the purpose of Cortex, not the full architecture.
